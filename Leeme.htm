Léeme del Servicio Web XML "pop3Auth"

Abstracto:
    pop3Auth es un Servicio Web XML que implementa la lógica e interfaces necesarias para validar credenciales de usuarios a través del servicio POP3 en uno o más servidores, es ideal para contar en sus aplicaciones con una forma de identificar usuarios que no dependa de un servicio de directorio o LDAP (Lightweight Directory Access Protocol) ó de la implementación de una base de datos personalizada con este fin.
    Este Servicio Web XML se ha hecho tanto com propósitos educativos como productivos y con la intención de que se pueda hospedar tanto en el servidor HTTP de sistemas operativos Microsoft Windows 2000 o superior (Internet Information Server) como en Apache para Linux de aquellas distribuciones para las cuales existan los correspondientes paquetes de instalación del proyecto Mono. Se ha programado utilizando Microsoft Visual Studio.Net 2003 con las librerías del .Net Framework versión 1.1.4322, pero es compatible también con la versión 2.0.50727, o sea que los paquetes de instalación del proyecto Mono necesarios para su funcionamiento en la plataforma Linux existen desde hace ya algún tiempo. Se ha probado tanto en sistemas operativos Microsoft Windows 2000 y 2003 como en las siguientes distribuciones de Linux: RedHat, CentOS, Mandrake y Fedora.
Herramientas y tecnologías para su desarrollo:
    Se ha utilizado Microsoft Visual Studio.Net 2003 como entorno integrado de desarrollo (IDE) y como lenguaje de programación C#, la librería utilizada ha sido el .Net Framework de Microsoft versión 1.1.4322. Por ser un Servicio Web XML que responde a la implementación para los Frameworks mencionados se emplean también las siguientes tecnologías: SOAP (Simple Object Access Protocol), XML (Extensible Markup Language) y HTTP (Hypertext Transfer Protocol) y TCP/IP (Transfer Control Protocol/Internet Protocol).
Componentes utilizados que se consideran ajenos a las tecnologías de los Servicios Web XML:
    Servidor POP (Post Office Protocol) según el RFC821.
Requerimientos:
    Sistema Operativo:
    • Cualquier sistema operativo para el cual exista un implementación el Microsoft .Net Framework, lo cual incluye las siguientes plataformas hasta ahora: Microsoft Windows, Linux, Solaris, Mac OS X, FreeBSD y Unix.
    Servidor HTTP:
    • Microsoft IIS (Internet Information Server).
    • Apache (para Linux).
    Librerías de software:
    • Microsoft .Net Framework 1.1.4322 ó superior, para hospedarse sobre Microsoft Windows.
    • Framework del proyecto Mono, para hospedarse sobre Apache para Linux.
    Otros:
    • Servidor de POP accesible mediante protocolo TCP/IP.

Implementación:
    Microsoft Internet Information Server:
    • Instalación de Microsoft .Net Framework 1.1.4322
    • Configuración de un "Directorio virtual" en el IIS cuya localización física almacenará los ficheros del Servicio Web XML pop3Auth.
    • Configuración, en el fichero Web.config que se incluye en el Servicio Web XML pop3Auth, del o los servidores de POP3 que se necesiten emplear.
    Apache para Linux:
    • Instalación de la correspondiente librería de Mono según la distribución de Linux empleada.
    • Configuración de un "Alias" para la aplicación en el fichero "httpd.conf". Ejemplo: <Alias /pop3Auth "/etc/httpd/XMLWebServices/pop3Auth">
    • Configuración, en el fichero Web.config que se incluye en el Servicio Web XML pop3Auth, del o los servidores de POP3 que se necesiten emplear.

Especificaciones del funcionamiento:
    Generales:
      El Servicio Web XML pop3Auth cuenta en su interface pública con dos métodos llamados "LogonToSpecificServer" y "LogonToServersInConfig", cada uno de ellos tiene el mismo objetivo, y devuelve el mismo tipo de datos para operaciones de identificación de usuario fallidas y satisfactorias pero reciben distinto número de parámetros como se muestra en las especificacines de cada uno de ellos a continuación.
    Interfaces pública:
      <LogonToSpecificServer>
        Parámetros de entrada:
        • POP3Server: una cadena que representa el nombre o la dirección IP de un servidor POP3 accesible que se va a utilizar para validar las credenciales del usuario que proporciona los datos a través de la aplicación que se considera cliente de este Servicio Web XML.
        • Port: un valor de tipo entero que representa el puerto por donde estará escuchando las peticiones el servidor POP3 en cuestión, normalmente el puerto 110.
        • userEmail: una cadena que representa la dirección electrónica del usuario que, haciendo uso de la aplicación que se considera cliente de este Servicio Web XML, está intentando ser identificado por el servidor POP3 en cuestión.
        • userPassword: una cadena que representa la contraseña del usuario que, haciendo uso de la aplicación que se considera cliente de este Servicio Web XML, está intentando ser identificado por el servidor POP3 en cuestión.
        Parámetros de salida:
        • True (verdadero): En caso de ser identificado el usuario de manera correcta.
        • False (falso): En caso de no estar correctas las credenciales del usuario.
        Excepciones (Errores):
          Nota.
          Las excepciones para este procedimiento se procesan a través de una clase personalizada llamada "pop3Exception" y se generaran bajo las siguientes circunstancias:
        • Imposible conectar con el servidor POP3 especificado.
        • Error al enviar el comando USER al servidor.
      <LogonToServersInConfig>
        Nota.
        En este caso el nombre del servidor y su respectivo puerto para el servicio POP3 se tomarán del fichero de configuración "Web.config" perteneciente a este Servicio Web XML.
        Parámetros de entrada:
        • userEmail: una cadena que representa la dirección electrónica del usuario que, haciendo uso de la aplicación que se considera cliente de este Servicio Web XML, está intentando ser identificado por el servidor POP3 en cuestión.
        • userPassword: una cadena que representa la contraseña del usuario que, haciendo uso de la aplicación que se considera cliente de este Servicio Web XML, está intentando ser identificado por el servidor POP3 en cuestión.
        Parámetros de salida:
        • True (verdadero): En caso de ser identificado el usuario de manera correcta.
        • False (falso): En caso de no estar correctas las credenciales del usuario.
        Excepciones (Errores):
          Nota.
          Las excepciones para este procedimiento se procesan a través de una clase personalizada llamada "pop3Exception" y también de otras clases propias del framework; se generaran bajo las siguientes circunstancias:
        • Imposible conectar con el servidor POP3 especificado.
        • Error al enviar el comando USER al servidor.
        • Errores en el fichero de configuración "Web.config" como son: la no existencia del a llave "pop3Servers" o la ausencia de valor para esta, la aparición malformada del nombre o dirección IP del servidor y/o del puerto para cualquiera de los servidores POP3 que se configuren en el fichero de configuración.
